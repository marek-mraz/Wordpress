{{ if .Values.sealedSecret.enabled -}}
IMPORTANT — SealedSecrets:
  sealedSecret.enabled is true. The following encrypted values must be populated
  in your values override file before deploying:

    sealedSecret.encryptedData.mariadb-root-password
    sealedSecret.encryptedData.mariadb-password
    sealedSecret.encryptedData.wordpress-password
    sealedSecret.encryptedData.oauth-client-secret

  Generate each value with:
    echo -n 'your-secret' | kubeseal --raw \
      --namespace <namespace> \
      --name {{ include "wordpress.fullname" . }}-credentials \
      --from-file=/dev/stdin

  If the encrypted values are empty, the Sealed Secrets controller will fail
  to decrypt and the pods will not be able to start.
{{- else }}
IMPORTANT — Plain Secrets:
  sealedSecret.enabled is false. Credentials are stored as a plain Kubernetes Secret.
  Ensure secret.mariadbRootPassword, secret.mariadbPassword, secret.wordpressPassword,
  and secret.oauthClientSecret are provided via --set or a git-ignored override file.
  NEVER commit real credentials to version control.
{{- end }}
